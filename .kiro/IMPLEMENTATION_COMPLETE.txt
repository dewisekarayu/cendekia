  ❌ No fallback strategies for role checking failures

  ❌ Manual authorization checks instead of policy

  ✅ Multiple fallback strategies (helper methods as backup)

  ✅ Policy-based authorization using Gate::inspect()
  ✅ Comprehensive logging & debugging

───────────────────────────────────────────────────────────────────────────────
🎓 LESSONS LEARNED
───────────────────────────────────────────────────────────────────────────────

1. Multi-Layer Authorization
   - Use multiple layers: middleware, controller, policy
   - Each layer serves different purpose
   - Defense in depth approach

2. Fallback Strategies
   - Never rely on single check
   - Provide multiple strategies
   - Handles unexpected cases

3. Logging & Debugging
   - Log at critical points
   - Include context (user_id, forum_id, etc.)
   - Helps troubleshoot production issues

4. Consistency
   - Same pattern for different roles
   - Makes code easier to maintain
   - Prevents bugs from duplication

5. Documentation
   - Document not just what but why
   - Include troubleshooting guides
   - Help future developers
